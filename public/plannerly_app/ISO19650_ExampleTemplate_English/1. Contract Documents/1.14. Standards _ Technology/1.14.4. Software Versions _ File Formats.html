File formats

To support OpenBIM workflows and long-term usability of information, project data shall be shared using both native and open formats where applicable. Native formats support authoring and coordination, while IFC and accessible formats support review, verification, and handover.

Why this matters

Defining software tools and file formats upfront prevents compatibility issues, supports OpenBIM workflows, and ensures information can be shared, reviewed, and verified consistently throughout the project lifecycle.
