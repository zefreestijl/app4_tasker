Information Requirements

To achieve the highest Task Team productivity we strive to be explicit in defining our Project Information Requirements and where possible in our Exchange Information Requirements we will define the:

  • Purpose - why we seek the information - for example to forecast replacement costs
  • Content - the breakdown of information - for example the geometrical, alphanumeric and documentation required
  • Form - how the information is presented - for example a schedule
  • Format - how the information is encoded - for example an .xlsx file type

Where Project and Exchange Information requirements are unclear it is expected that the delivery team will seek clarification by asking questions and requesting explanations through the commenting features inside Plannerly.
