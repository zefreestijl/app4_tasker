BCF Issue Management

One of the key aspects of successful BIM management is effective issue management. To facilitate this, we will use an open standard for managing issues on a BIM project called BCF (BIM Collaboration Format).

BCF allows project team members to identify and communicate issues related to the project model in a structured way. This includes the ability to assign issues to specific team members, add comments and attachments, and track the status of issues over time.

The BCF-API (Application Programming Interface) supports the exchange of BCF issues between software applications via a RESTful web interface. This means that different software tools used by different project team members can communicate and share issue information seamlessly.

Our project will use the BCF format for issue management to help streamline the project management process, reduce errors and omissions, and improve communication and collaboration between team members.

Example BCF Issue Management Workflow

Step Action Responsible Party Status
1 Identify an issue in the project model Designer Open
2 Create a BCF issue and assign it to the responsible party Project Manager Open
3 Responsible party receives notification of issue and reviews it Contractor Open
4 Responsible party adds comments and attachments to the issue Contractor In Progress
5 Responsible party resolves the issue and marks it as closed Contractor Closed
